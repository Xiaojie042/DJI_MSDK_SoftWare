# DJI 无人机实时监控系统

本项目包含基于 Vue 3 + FastAPI 驱动的无人机监控指挥中心，用于实时接收 DJI 后台 MSDK 数据兵展示坐标、航向、等遥测数据。

## 📍 架构与端口说明

本系统包含两个核心网络端口，**请仔细区分（非常重要）**：

1. **TCP 数据接收端口 (默认 `9001`)**
   - **用途**：这是为 `DJI遥控器` (Android端网络接口) 预留的TCP原始Socket服务器。
   - **连接行为**：遥控器应用需要将数据发送至该主机的 `9001` 端口。只有连接此处，系统才会解析收到的字节流或 JSON 并产生飞行数据。
   
2. **HTTP / WebSocket API 端口 (默认更改为 `8888`)**
   - **用途**：这是 `FastAPI` 和 `Vue前端` 通信与访问的端口。包含了页面访问与 WebSocket 数据实时下发。
   - **注意**：遥控器**不应该**连接 8888 端口。

## 🚀 本地/生产环境运行指南 (支持夸电脑连按)

如果您想在另一台电脑、手机浏览器或者遥控器（作为不同设备）上运行/查看此系统，请确保**所有设备处于同一个局域网，并且关闭防火墙对 `8888` 与 `9001` 端口的拦截** (如 Windows Defender)。

### 1. 运行后端 (FastAPI)
```bash
cd backend
conda activate drone-monitor
# 使用指定的 8888 端口进行启动绑定，--host 0.0.0.0 表示允许所有内网设备连接
uvicorn app.main:app --host 0.0.0.0 --port 8888
```

### 2. 运行前端 (Vue)
前端为了可以在其他电脑查看，开发环境需附加 `--host` 标志：
```bash
cd frontend
# (可选) 若使用 npm 请在 package.json 中的 dev 脚本加上 --host，即 "vite --host"
npm run dev -- --host
```
此时终端会打印出一个 `Network: http://192.168.x.x:5173/` 这样的地址。在任意同局域网设备打开这个IP即可进入监控大屏界面。

### 3. DJI遥控器连接步骤
1. 确保遥控器连接的 WiFi 与当前运行后端的电脑是**同一个网段**。
2. 查找本机的局域网 IP（在终端输入 `ipconfig` 获取 IPv4 地址，如 `192.168.1.100`）。
3. 在 DJI MSDK App 中的 Socket 配置中填写：
   - 目的 IP：`192.168.1.100` （填你真实的IP）
   - 目的 端口：`9001`
4. 发起连接开始传输，此时后端 FastAPI 日志中应该会有如下输出：
   `INFO     TCP 客户端已连接 peer=('192.168.1.xxx', 53421) total_clients=1`

### 4. 模拟器调试
若暂无遥控器硬件可用，只需另开一个命令行运行本脚本。它会朝 `127.0.0.1:9001` 发射数据并展示在本地大屏。
```bash
cd backend
conda activate drone-monitor
python simulate_drone.py
```

## ❓ 常见问题排错 (FAQ)

**Q1：MQTT 服务器如果没有建立，会影响前端数据接收吗？**
A：**完全不会！** 本系统的 `DataDispatcher` (数据分发) 使用了异步非阻塞设计，即便向 MQTT 抛出数据时抛弃报错或跳过（因为未连接），并不会阻塞同一进程内对 WebSocket（前端组件） 下发指令。不运行 Docker MQTT 也能顺利呈现所有界面变化！

**Q2：遥控器明明连接了却完全没有提示日志？**
A：首先确认连接端口是 `TCP 9001` 而不是 `8888` 或 `8000`。其次若跨设备发送，请排查 Windows 电脑防火墙是否放行了 Python 网络权限。

**Q3：前端一直显示 "Server Disconnected"（服务器失联）？**
A：原先我们因为 `useWebSocket.js` 代码中写死了 8000 端口，当您把 Uvicorn 切至 8888 时无法衔接。现已重构动态感知功能：无论何种端口部署，大屏只要加载，均向同地址主机的 `8888` 发起握手！刷新前端页面即可变绿。
